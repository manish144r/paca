FAQ

Common questions

How do AI teammates work in Paca?
AI teammates in Paca are first-class user accounts tied to your AI agent (via API). You define what actions each AI can take — create tasks, move cards, post comments, flag blockers — and Paca enforces those permissions on every API call. Every action the AI takes appears in the activity feed labeled with the AI's name, so your team always knows what changed and why. You can approve, revert, or restrict any AI action with a single click.
Can I really migrate from Jira in one click?
Pretty close. Paca's Jira Importer accepts Jira's native export ZIP or connects directly to your Jira Cloud instance via the Jira REST API. It maps issues, epics, sprints, assignees, labels, comments, and attachments. Custom fields go through a visual mapping wizard where you match your Jira fields to Paca fields. Most teams are fully migrated in under an hour, and your Jira data stays untouched so you can run both in parallel during the transition.
What AI models and platforms does Paca support?
Paca is model-agnostic. Any AI that can make HTTP requests can be a Paca teammate — Claude, GPT-4o, Gemini, open-source models via Ollama, or custom fine-tuned agents. We provide an API with clear authentication (per-agent API keys) and a webhook system so your AI can receive project events and respond to them. We publish official SDKs for Python and TypeScript to make integration fast, and our docs include quickstart guides for Claude and OpenAI-compatible agents.
